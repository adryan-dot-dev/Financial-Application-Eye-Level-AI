## מה זה ולמה

### הרעיון

במקום סוכן אחד שעושה הכל, יש לך **צוות של 5 סוכנים מומחים**:

```
┌─────────────────────────────────────────────────────────────┐
│                         אתה                                 │
│                    (מנהל הפרויקט)                           │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                    ORCHESTRATOR                             │
│              (ארכיטקט + מנהל משימות)                         │
│                                                             │
│   • מקבל ממך דרישות                                         │
│   • מתכנן ארכיטקטורה                                        │
│   • מחלק משימות לסוכנים                                     │
│   • עוקב אחרי התקדמות                                       │
└─────────────────────────┬───────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┬─────────────────┐
        ▼                 ▼                 ▼                 ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ IMPLEMENTER │    │   TESTER    │    │  REVIEWER   │    │    DOCS     │
│             │    │             │    │             │    │             │
│ כותב קוד    │    │ כותב טסטים  │    │ בודק איכות  │    │ כותב תיעוד  │
│ מתקן באגים  │    │ מריץ בדיקות │    │ refactoring │    │ README, API │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### למה זה עובד

| בעיה עם סוכן אחד | פתרון עם צוות |
|------------------|---------------|
| מתבלבל בין תפקידים | כל סוכן ממוקד במשימה אחת |
| שוכח הקשר בשיחות ארוכות | הקשר שמור בקבצים |
| קשה לעקוב אחרי התקדמות | סטטוס ברור לכל משימה |
| אין בקרת איכות | סוכן נפרד לבדיקות |

## הסוכנים

### Tab 1: ORCHESTRATOR (הארכיטקט)

**תפקיד:** מתכנן ומנהל - לא נוגע בקוד

**מה הוא עושה:**
- מקבל ממך דרישות ושואל שאלות הבהרה
- מתכנן את הארכיטקטורה
- מפרק למשימות ברורות
- כותב משימות לסוכנים האחרים
- עוקב אחרי סטטוסים ומתאם

**מתי מדברים איתו:**
- בתחילת פרויקט
- כשרוצים לשנות כיוון
- כשצריך להוסיף פיצ'ר
- כשיש בעיה שצריך להחליט עליה

---

### Tab 2: IMPLEMENTER (המפתח)

**תפקיד:** כותב קוד - לא מתכנן ולא בודק

**מה הוא עושה:**
- קורא משימות מ-`agents/tasks/implement.md`
- כותב קוד לפי ההנחיות
- מעדכן סטטוס ב-`agents/status/implement.md`

**מתי מדברים איתו:**
- כשיש לו משימה חדשה
- כשהוא נתקע וצריך עזרה

---

### Tab 3: TESTER (הבודק)

**תפקיד:** כותב ומריץ טסטים - לא מתקן באגים

**מה הוא עושה:**
- קורא משימות מ-`agents/tasks/test.md`
- כותב unit tests ו-integration tests
- מריץ את הטסטים ומדווח תוצאות
- מזהה באגים ומתעד אותם

**מתי מדברים איתו:**
- כשיש קוד חדש לבדוק
- כשצריך להריץ את כל הטסטים

---

### Tab 4: REVIEWER (בקרת איכות)

**תפקיד:** סוקר ומשפר קוד

**מה הוא עושה:**
- קורא משימות מ-`agents/tasks/review.md`
- עושה code review
- מזהה בעיות (security, performance, style)
- מבצע refactoring כשצריך

**מתי מדברים איתו:**
- כשיש קוד מוכן לסקירה
- כשצריך לשפר קוד קיים

---

### Tab 5: DOCS (התיעוד)

**תפקיד:** כותב תיעוד

**מה הוא עושה:**
- קורא משימות מ-`agents/tasks/docs.md`
- כותב README
- מתעד API
- מוסיף הערות לקוד מורכב

**מתי מדברים איתו:**
- כשיש קוד חדש שצריך תיעוד
- כשצריך לעדכן README

## פתיחת פרויקט חדש

### שלב 1: העתק את התבנית

```bash
cp -r multi-agent-template my-new-project
cd my-new-project
```

### שלב 2: פתח 5 טאבים של Claude Code

פתח את הפרויקט ב-Claude Code ופתח 5 טאבים/חלונות.

### שלב 3: אתחל כל סוכן

**Tab 1 - הדבק:**
```
אתה Orchestrator. קרא agents/ORCHESTRATOR.md והתחל לעבוד.
```

**Tab 2 - הדבק:**
```
אתה Implementer. קרא agents/IMPLEMENTER.md
```

**Tab 3 - הדבק:**
```
אתה Tester. קרא agents/TESTER.md
```

**Tab 4 - הדבק:**
```
אתה Reviewer. קרא agents/REVIEWER.md
```

**Tab 5 - הדבק:**
```
אתה Docs. קרא agents/DOCS.md
```

## זרימת עבודה יומיומית

### תרשים זרימה

```
┌─────────────────────────────────────────────────────────────┐
│  1. אתה → Orchestrator: "רוצה לבנות X"                      │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  2. Orchestrator: מתכנן וכותב משימות לקבצים                  │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  3. אתה → Implementer: "יש לך משימה"                        │
│     Implementer: עובד ומעדכן סטטוס                          │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  4. אתה → Orchestrator: "מה הסטטוס?"                        │
│     Orchestrator: בודק וכותב משימות ל-Tester                │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  5. אתה → Tester: "יש לך משימה"                             │
│     Tester: כותב טסטים ומדווח                               │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  6. אתה → Orchestrator: "מה הסטטוס?"                        │
│     ... וכן הלאה עד שהפרויקט מוכן                           │
└─────────────────────────────────────────────────────────────┘
```

### דוגמה מעשית

**אתה (Tab 1):**
> "רוצה לבנות API לניהול משימות עם CRUD"

**Orchestrator:**
> "מעולה. אני מתכנן את זה ככה:
> 1. מודל Task עם: id, title, done, created_at
> 2. API endpoints: GET/POST/PUT/DELETE
> 3. SQLite לדאטאבייס
> 
> כתבתי משימה ל-Implementer. לך ל-Tab 2 ותגיד לו לעבוד."

**אתה (Tab 2):**
> "יש לך משימה חדשה"

**Implementer:**
> "קראתי את המשימה. אני יוצר:
> - src/models/task.py
> - src/routes/tasks.py
> - src/db.py
> 
> סיימתי ועדכנתי את הסטטוס."

**אתה (Tab 1):**
> "המפתח סיים"

**Orchestrator:**
> "ראיתי. כתבתי משימה ל-Tester. לך ל-Tab 3."

## פקודות מהירות

### לכל הסוכנים

| מה לומר | מתי |
|---------|-----|
| `יש לך משימה חדשה` | כשיש משימה בקובץ שלו |
| `תמשיך לעבוד` | כשהפסיק באמצע |
| `מה הסטטוס שלך?` | לבדוק התקדמות |
| `תקרא שוב את הקובץ שלך` | אם נראה מבולבל |

### ל-Orchestrator בלבד

| מה לומר | מתי |
|---------|-----|
| `מה הסטטוס הכללי?` | לקבל תמונה מלאה |
| `רוצה לשנות...` | לשנות כיוון |
| `תוסיף פיצ'ר...` | להוסיף משהו |
| `יש בעיה עם...` | לדווח על בעיה |
| `מה השלב הבא?` | כשלא בטוח מה לעשות |

## פתרון בעיות

### סוכן שוכח מי הוא

**בעיה:** סוכן מתחיל להתנהג כמו סוכן אחר

**פתרון:**
```
תקרא שוב את agents/[AGENT].md ותעבוד לפי ההוראות שם
```

---

### סוכן לא רואה משימות

**בעיה:** "אין לי משימות"

**פתרון:**
1. בדוק שה-Orchestrator באמת כתב משימה
2. תגיד לסוכן: `תקרא את agents/tasks/[agent].md`

---

### התנגשות בין סוכנים

**בעיה:** שני סוכנים עובדים על אותו קובץ

**פתרון:**
1. עצור את שניהם
2. לך ל-Orchestrator ותגיד: "יש התנגשות בין X ל-Y"
3. הוא יתאם מחדש

---

### משימה לא ברורה

**בעיה:** סוכן לא מבין מה לעשות

**פתרון:**
1. הסוכן צריך לכתוב ב-`agents/status/[agent].md`: "לא ברור - צריך הבהרה"
2. לך ל-Orchestrator: "ל-X לא ברורה המשימה"
3. Orchestrator יבהיר

---

### שיחה ארוכה מדי

**בעיה:** הסוכן מתחיל לשכוח

**פתרון:**
1. סגור את הטאב
2. פתח מחדש
3. הדבק את הפרומפט ההתחלתי
4. תגיד: "יש לך משימות בקובץ, תמשיך מאיפה שהפסקת"

## מתי להשתמש ב-Claude Code Architect

### מה זה Claude Code Architect?

זה **הפרויקט הנוכחי שאנחנו בו עכשיו** - כאן אתה מקבל:
- תכנון ארכיטקטורה לפני שמתחילים
- עזרה בבחירת טכנולוגיות
- הכנת פרומפטים מדויקים
- פתרון בעיות ארכיטקטוניות

### מתי לבוא לכאן (Claude Code Architect)?

| מצב | לאן ללכת |
|-----|----------|
| רוצה להתחיל פרויקט חדש מאפס | **לכאן** - לתכנון |
| לא בטוח איזה טכנולוגיה לבחור | **לכאן** - לייעוץ |
| רוצה לשפר את שיטת העבודה | **לכאן** - לשדרוג |
| מתלבט בין כמה גישות | **לכאן** - לניתוח |
| צריך לתכנן אינטגרציה מורכבת | **לכאן** - לארכיטקטורה |

### מתי ללכת ישר ל-Claude Code?

| מצב | לאן ללכת |
|-----|----------|
| יש לך כבר תכנית ברורה | **Claude Code** - לביצוע |
| משימה פשוטה (תיקון באג) | **Claude Code** - לביצוע |
| המשך עבודה על פרויקט קיים | **Claude Code** - לביצוע |
| יודע בדיוק מה צריך | **Claude Code** - לביצוע |
